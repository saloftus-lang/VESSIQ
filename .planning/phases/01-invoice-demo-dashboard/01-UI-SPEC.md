# Phase 1 — UI Design Contract: Invoice Demo Dashboard

> Visual and interaction contract for `dashboard.html`. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This is a standalone HTML file — no build step, no component library, no shadcn. All design decisions are expressed as CSS custom properties and vanilla CSS.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — vanilla CSS with CSS custom properties |
| Preset | not applicable |
| Component library | none — hand-rolled HTML/CSS patterns |
| Icon library | Unicode symbols + inline SVG (no icon CDN) |
| Font (primary) | Inter — must be self-hosted or inlined (see DS-05 note below) |
| Font (monospace) | JetBrains Mono — must be self-hosted or inlined (see DS-05 note below) |
| Font fallback | `-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif` (renders immediately before font loads) |
| Charts | ApexCharts 3.54.1 via jsDelivr CDN — pinned exact version, never `@latest` |
| Animation | GSAP 3.12.5 core via jsDelivr CDN — UMD bundle (not ESM), works over `file://` if needed |
| Rendering target | Desktop only — 1280×800 minimum, 1440×900 optimized |

**DS-05 Font Loading Note:** REQUIREMENTS.md DS-05 requires offline-safe rendering. Google Fonts CDN violates this. Inter and JetBrains Mono must be bundled locally (downloaded and placed in `/fonts/`, referenced via `@font-face` with `font-display: swap`). This satisfies both DS-05 and eliminates the CDN round-trip. The CDN `<link>` tags in the CDN Stack section are retained as fallback documentation only — the executor must replace them with self-hosted `@font-face` declarations.

Source: REQUIREMENTS.md DS-05, DS-06, DS-07; RESEARCH.md stack.md sections 2, 4, 5

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge inner padding, inline chip spacing |
| sm | 8px | Button inner padding (vertical), filter pill gap, badge horizontal padding |
| md | 16px | Card inner padding (compact), table cell padding, form input padding |
| lg | 24px | Card inner padding (standard), grid gap between cards, section sub-spacing |
| xl | 32px | Section header margin-bottom, chart container padding, ROI panel padding |
| 2xl | 48px | Header horizontal padding, content area horizontal padding, major section breaks |
| 3xl | 64px | Page-level top padding (hero area) |

Exceptions:
- KPI tile touch areas: minimum 44px height (accessibility, even though desktop-only)
- Table row height: 52px (ensures legibility of two-line data rows at 14px body)
- Expandable row content: 16px padding top/bottom inside the expanded area
- Header height: 68px fixed

Additional typography rules:
- KPI tile labels: uppercase, letter-spacing 0.5px (pattern from existing demo)

## Component Inventory

These are the UI components needed for Phase 1. Each maps to one or more requirements.

### 2. KPI Tile Row
- Layout: 4-column grid, `gap: 24px`, full width of content area
- Each tile: glassmorphism card, 24px padding, min-height 120px
- Tile structure: label (11px uppercase caption) → number (36px display, weight 700) → delta badge (optional)
- Numbers animate on page load with GSAP `countTo` — duration 1.2s, ease `power2.out`, triggered by `DOMContentLoaded`
- Tile 1 "Total Invoices Audited": number color `--white`, value 847
- Tile 2 "Total Overcharges Found": number color `--red`, value 288 (count of flagged invoices)

- **Primary visual anchor:** KPI tile row — animated counter numbers on load draw the viewer's eye first and establish the value proposition before any other element is read.

## Interaction Contracts

### Filter State Machine

```
state = {
  selectedCarrier: 'ALL',   // 'ALL' | 'MAEU' | 'MSCU' | 'EGLV' | 'HLCU' | 'PSHA' | 'BNSF'
  dateRange: '3M',          // '1M' | '3M' | 'YTD'
  expandedRowId: null,      // string invoice ID or null
  roiInput: null            // number or null
}
```

- All filter changes call `setState(patch)` then `render()`
- `render()` calls `renderTable()`, `renderKPIs()`, `renderCharts()` in sequence
- Charts call `chart.updateSeries()` — never re-initialize ApexCharts after first render
- KPI totals are derived from filtered data, not from constants

### Row Expand / Collapse

- Click row → if `state.expandedRowId === row.id` then collapse (set null), else expand (set row.id)
- Use event delegation: single `addEventListener('click', handler)` on `<tbody>`, read `event.target.closest('tr[data-invoice-id]')`
- Expanded content injected as next sibling `<tr>` with class `row-detail`
- Animation: `max-height: 0` → `max-height: 400px` CSS transition, `overflow: hidden`, duration 0.25s ease
- On expand: set chevron `aria-label="Collapse invoice details"`; on collapse: set chevron `aria-label="Expand invoice details"`

### GSAP Counter Animation

- Triggered on `DOMContentLoaded`
- Stagger: `gsap.from('.kpi-tile', { y: 20, opacity: 0, stagger: 0.08, duration: 0.5 })`
- Counter: `gsap.to(counterEl, { innerHTML: targetValue, duration: 1.2, ease: 'power2.out', snap: { innerHTML: 1 }, onUpdate: formatValue })`
- Dollar amounts: formatted with `$` prefix and comma-separation during animation
- Percentage amounts: formatted with `%` suffix

### ApexCharts Initialization Rules

- Initialize inside `document.addEventListener('DOMContentLoaded', ...)` only
- Always set `theme: { mode: 'dark' }` and `chart: { background: 'transparent' }` on init
- Chart container `<div>` must have `background: transparent !important` in CSS
- Never build a theme toggle — dark only
- Update with `chart.updateSeries()` never `chart.destroy()` + reinitialize

### ROI Calculator

- Event: `oninput` on the dollar input field
- Strip non-numeric characters before calculating
- Formula:
  - Annual recovery = `(input * 12) * 0.035`
  - VESSIQ fee = `annual_recovery * 0.25`
  - Net benefit = `annual_recovery - fee`
- Format all outputs as `$X,XXX,XXX` using `toLocaleString('en-US', { style: 'currency', currency: 'USD', maximumFractionDigits: 0 })`
- If input is empty or 0: show `$—` in all three output tiles

## Maritime Data Credibility Checklist

All fake data baked into the `DATA` const must pass this checklist before any demo. Non-negotiable for credibility with logistics buyers.

| Check | Requirement | Values to use |
|-------|-------------|---------------|
| SCAC codes | Valid 4-letter codes only | MAEU (Maersk), MSCU (MSC), EGLV (Evergreen), HLCU (Hapag-Lloyd), PSHA (Pasha Hawaii), BNSF (BNSF rail) |
| UN/LOCODEs | 5-char format: 2-letter country + 3-letter city | USLAX, USLGB, USSEA, USHON, CNSHA, SGSIN, DEHAM |
| Container numbers | ISO 6346: 4-letter owner prefix + U/J/Z + 6 digits + check digit | MAEU1234567, MSCU7654321 |
| Invoice amounts | Non-round, with cents | $12,847.34 not $12,000 |
| Invoice numbers | Carrier-realistic formats | Maersk: `MAEU-2024-XXXXX`; MSC: `MSC-XXXXXXX` |
| Voyage numbers | 3-letter service + 3-digit voyage | `AE-1 024E`, `SEAS024` |
| Charge types | Real surcharge names | BAF, THC, PSS, D&D, PRS (Peak Season Surcharge) |
| Port combinations | Realistic trade lanes | CNSHA→USLAX, SGSIN→USLGB, DEHAM→USNYK |
| Dollar ranges | Per-TEU realistic | $3,000–$8,000/TEU ocean; $2,500–$5,000 rail |
| Overcharge ranges | 2–15% of invoice value | Not >20% — signals implausibility to finance buyers |

Source: REQUIREMENTS.md REL-05; RESEARCH.md pitfalls.md; features.md data section

```css
@font-face {

@font-face {

  font-display: swap;
}
```

## Known Bugs and Mitigations

| Bug | Severity | Mitigation |
|-----|----------|-----------|
| ApexCharts ghost background on dark mode (#4028, #3387) | High | Hardcode `theme: { mode: 'dark' }` + `chart: { background: 'transparent' }` on init; add `background: transparent !important` to chart container CSS; never build a theme toggle |
| Safari `backdrop-filter` requires `-webkit-` prefix | Medium | Always declare both `backdrop-filter` and `-webkit-backdrop-filter`; always declare a non-transparent `background-color` as fallback |
| `fetch()` blocked over `file://` protocol | High | Serve via `python3 -m http.server` or equivalent — document in VESSEL README; no `fetch()` calls in demo anyway (all data is baked in) |

Source: STATE.md blockers; RESEARCH.md pitfalls.md
