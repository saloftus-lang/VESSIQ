```css

```

Safari fallback — always declare a non-transparent background-color before the rgba glass fill:
```css
background-color: #132B4E;  /* must be declared first, before rgba() override */
background: rgba(255,255,255,0.05);
```

No CSS 3D perspective transforms or hover tilt on any card in this phase (CONTEXT.md D-02). KPI tiles use `translateY(-3px)` hover lift only, no `rotateX`.

Performance constraint: No more than 7 glassmorphism elements visible in any single viewport. This phase has: 3 KPI tiles + 2 section widgets (tasks, notes) + 3 benchmark cards = 8 total but they do not all appear in the same viewport window at 1440×900. Acceptable.

**State and persistence:**
- State: array of `{id: string, text: string, done: boolean}`
- IDs: `'t' + Date.now()` for new tasks
- Persistence key: `localStorage['fnd-tasks']` — serialize with `JSON.stringify`, parse with `JSON.parse`
- Load on `DOMContentLoaded`: if key absent, use pre-seeded array; if present, parse and render
- Save on every mutation (add, toggle done, delete): call `localStorage.setItem('fnd-tasks', JSON.stringify(state.tasks))`

Requirement: FND-03

**For metrics where higher = better (Dispute Win Rate):**

## Interaction Contracts

### State Model

```javascript
const state = {
  tasks: [],     // array of { id: string, text: string, done: boolean }
  notes: ''      // string — full textarea contents
}
```

- `setState(patch)` merges patch into state, then calls `renderTasks()` or `renderNotes()` as appropriate, then persists to localStorage
- Tasks and notes are independent — updating one does not re-render the other

### Task Interactions

| Action | Trigger | Behavior |
|--------|---------|---------|
| Add task | Click "Add" button OR press Enter in input | Trim input; if empty, do nothing; else push `{ id: 't' + Date.now(), text, done: false }` to state.tasks; clear input; re-render; persist |
| Toggle done | Click checkbox | Flip `task.done`; re-render; persist |
| Delete task | Click × button | Splice task from array; re-render; persist |
| Load on init | `DOMContentLoaded` | `JSON.parse(localStorage.getItem('fnd-tasks'))` — if null, use pre-seed array; set state.tasks; render |

### Notes Interactions

| Action | Trigger | Behavior |
|--------|---------|---------|
| Auto-save | `input` event on textarea | `localStorage.setItem('fnd-notes', textarea.value)` — no state update needed, no re-render |
| Load on init | `DOMContentLoaded` | `textarea.value = localStorage.getItem('fnd-notes') \|\| ''` |

### GSAP Counter Animation

- Triggered on `DOMContentLoaded`, after localStorage load and initial render
- Entrance stagger: `gsap.from('.kpi-tile', { y: 20, opacity: 0, stagger: 0.08, duration: 0.5, ease: 'power2.out' })`
- Counter for integers (tiles 1 and 2): `gsap.to(obj, { v: targetInt, duration: 1.2, ease: 'power2.out', snap: { v: 1 }, onUpdate: () => el.textContent = Math.round(obj.v) })`
- Counter for dollar amount (tile 3): `gsap.to(obj, { v: 187500, duration: 1.2, ease: 'power2.out', snap: { v: 1 }, onUpdate: () => el.textContent = '$' + Math.round(obj.v).toLocaleString('en-US') })`
- Benchmark cards do not animate — static display only

### LocalStorage Keys

| Key | Type | Default |
|-----|------|---------|
| `'fnd-tasks'` | JSON string | Pre-seeded 3-task array (first load only) |

## Known Bugs and Mitigations

| Bug | Severity | Mitigation |
|-----|----------|------------|
| Safari `backdrop-filter` requires `-webkit-` prefix | Medium | Always declare both `backdrop-filter` and `-webkit-backdrop-filter`; always declare `background-color: #132B4E` before the `rgba()` glass fill |
| Fonts fail over `file://` protocol | High | Must be served via `python3 -m http.server` — same requirement as `dashboard.html` |
| localStorage unavailable in private browsing (some browsers) | Low | Wrap localStorage calls in try/catch; if unavailable, app still works — state just does not persist |
