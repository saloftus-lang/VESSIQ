Unified Schema Normalization

Raw invoices are normalized into a unified maritime event schema. This creates a "single source of truth" across all carriers and terminals, enabling consistent auditing rules.

Before (Raw Data)

Maersk Invoice: BKG#, ContainerID, BaseFreight, THC_Port, CFS_Fee, Bunker, Tax...

HAPAG Invoice: BookingNo, Container, OceanFreight, PortCharges, CFS, Surcharge...

Different field names. No consistency. Errors slip through.

After (Normalized)

All Carriers:

- vessel_name, voyage_number
- container_number, booking_number
- base_freight, terminal_handling_charge
- origin_port, destination_port
- total_invoice_amount, currency

Consistent fields. Audit rules work universally.

Result: One audit engine. Unlimited carriers. No custom logic per vendor.
